\subsection{Hierarchical Topology Failures (9 partial results)}

All 9 hierarchical failures occurred on high-I tasks ($I \geq 0.8$). The consistent pattern: decomposition boundaries fail to capture cross-cutting concerns. When a problem requires holistic understanding (e.g., detecting race conditions across functions), splitting by function misses emergent interaction bugs.

\subsection{Debate Topology Failures (7 partial results)}

Debate's failures cluster around merge inconsistency: two agents produce independently valid solutions with divergent design choices, and the resolution step introduces integration errors. On high-D tasks ($D \geq 0.8$), debate achieved only 33\% accuracy because merging two complete but architecturally incompatible solutions is harder than building one from a shared plan.

\textbf{Tools available to all topologies:} Read, Write, Edit, Bash, Grep, Glob, WebSearch, WebFetch --- identical across all topologies, ensuring performance differences arise from orchestration strategy.

\textbf{Topology configurations:} Flat = single-agent ReAct loop. Hierarchical = planner agent decomposes task, spawns executor agents. Debate = two independent agents solve in parallel, results compared and reconciled by judge.

\textbf{Evaluation:} Single human evaluator scoring correct/partial/incorrect per task-topology pair. Correct = fully matches expected answer. Partial = correct approach with minor errors. Incorrect = fundamentally wrong or incomplete.

\textbf{Limitations of reproducibility:} (1) Non-deterministic generation: LLM outputs are stochastic; re-running produces different outputs. (2) Temporal dependence: research tasks query live web data as of March 15, 2026. (3) Evaluator subjectivity: the correct/partial/incorrect rubric involves human judgment.

\textbf{Artifacts released:} All 82 task prompts (JSON), expected answers, raw results per topology, D-I-T annotations, and figure generation scripts.
